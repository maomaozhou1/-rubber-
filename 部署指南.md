# **橡胶黏弹性实验室：部署指南**

本指南将引导您将“橡胶黏弹性实验室”程序部署到互联网，以便他人访问。由于该项目是基于 **Vite + React** 的单页面应用（SPA），生成的 `dist` 文件夹是静态的，因此部署非常灵活。

---

## **1. 本地构建 (准备阶段)**
在部署之前，请确保本地构建成功：
```bash
npm install
npm run build
```
这将在项目根目录下生成 `dist` 文件夹。

---

## **2. 方案一：Vercel / Netlify (推荐，最简单)**
这是最推荐的方案，支持 GitHub 自动同步，且免费额度足够个人使用。

### **操作步骤 (以 Vercel 为例)**：
1. **上传代码**：将代码推送到 GitHub、GitLab 或 Bitbucket 仓库。
2. **连接项目**：登录 [Vercel](https://vercel.com/)，点击 **Add New Project**。
3. **选择仓库**：导入您的项目仓库。
4. **配置参数**：
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Environment Variables**: 如果您使用了 Gemini AI 功能，请在此添加 `GEMINI_API_KEY`。
5. **点击 Deploy**：几分钟后即可通过生成的域名访问。

---

## **3. 方案二：GitHub Pages (适合个人主页)**
如果您希望直接部署在 GitHub 上。

### **操作步骤**：
1. **安装插件**：
   ```bash
   npm install gh-pages --save-dev
   ```
2. **修改 `package.json`**：
   添加 `"homepage": "https://<your-username>.github.io/<repo-name>/"`
   在 `scripts` 中添加：
   ```json
   "predeploy": "npm run build",
   "deploy": "gh-pages -d dist"
   ```
3. **修改 `vite.config.ts`**：
   在 `defineConfig` 中添加 `base: '/<repo-name>/'`。
4. **运行部署**：
   ```bash
   npm run deploy
   ```

---

## **4. 方案三：传统服务器 (Nginx / Docker)**
适合在自有服务器或云服务器上部署。

### **Nginx 配置示例**：
将 `dist` 文件夹上传到服务器路径（如 `/var/www/rubber-vis/`），并配置 Nginx：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/rubber-vis;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### **Docker 方案**：
在根目录创建 `Dockerfile`：
```dockerfile
# 阶段1: 构建
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# 阶段2: 服务
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```
然后运行：
```bash
docker build -t rubber-vis .
docker run -d -p 8080:80 rubber-vis
```

---

## **5. 注意事项**
- **环境变量**：如果在 `vite.config.ts` 中使用了 `GEMINI_API_KEY`，请务必在部署平台的 **Settings -> Environment Variables** 中设置对应的 API Key。
- **域名加速**：如果访问缓慢，建议开启 CDN 加速（Vercel/Netlify 已默认开启）。
